# AGENTS.md

このリポジトリで作業するエージェントは、KP野帳の今後の開発を安定させるため、以下のルールを必ず守ってください。

## 作業開始前の準備

1. 作業を始める前に、必ず最新の `main` を取得してください。
2. `origin` が未設定の場合は、次のリポジトリを `origin` として設定してください。

   ```bash
   git remote add origin https://github.com/kanna19840131/kp-yacho.git
   ```

3. 最新の `main` を取得できなかった場合は、実装を開始しないでください。その場合は、取得できなかった理由を報告してください。
4. 最新の `main` を基準に、新しい作業用ブランチを作成してから作業してください。

## 実装時の注意

5. 既存機能を壊さないようにしてください。
6. `localStorage` のキー `kp_yacho_v12` に保存しているデータ形式を変更する場合は、過去データとの互換性を必ず維持してください。
7. KP野帳は Android スマートフォンで使うことを前提に、画面表示や操作性を確認してください。

## 実装後の確認

8. 実装後は、既存機能と追加機能の両方をテストしてください。
9. テスト結果を報告してから、`commit` と PR 作成を行ってください。
